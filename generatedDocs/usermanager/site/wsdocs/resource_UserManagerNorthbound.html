UserManagerNorthbound

This class provides REST APIs to manage users. This API will only be availalbe via HTTPS.

Authentication scheme : HTTP Basic
Authentication realm : opendaylight
Transport : HTTPS

HTTPS Authentication is disabled by default so to use UserManager APIs turn on HTTPS on Web Server

The following resources are part of this group:

  • /users
  • /users/{userName}

/users

Mount Point: /controller/nb/v2/usermanager/users

POST

Add a user

Request Body

element: userConfig
media types: application/xml
application/json

the UserConfig user config structure in request body

Response Body

element: (custom)
media types: */*
application/xml

Response as dictated by the HTTP Response Status code 

Example:

Request URL:
https://localhost/controller/nb/v2/usermanager/users

Request body in XML:
<userConfig>
<user>testuser</user>
<roles>Network-Admin</roles>
<password>pass!23</password>
</userConfig>

Request body in JSON:
{
"user":"testuser",
"password":"pass!23",
"roles":[
"Network-Admin"
]
}

Status Codes

code description
201 User created successfully
400 Invalid data passed
401 User not authorized to perform this operation
409 User name in url conflicts with name in request body
404 User config is null
500 Internal Server Error: Addition of user failed
503 Service unavailable
